# Phase 1c: 経路記録

## このフェーズで入った概念(1つだけ)

**「マーカー基準の座標で歩いた軌跡をポリラインとして保存する」**

1bで手に入れた「起動をまたいで同じ意味を持つ座標系(マーカー座標系)」の上に、歩いた軌跡を点列(ポリライン)として記録・保存する。保存された経路は翌日以降のセッションでもマーカー再認識だけで有効になる。

## 記録の仕組み

- サンプリングは**距離ベース**: 前回記録点から0.3m以上移動したら1点追加。時間ベースにしない理由は、静止中に同じ場所の点が無限に増えるのを防ぐため
- 各点はマーカー座標系の x/y/z と記録開始からの経過ミリ秒を持つ
- カメラのトラッキングがPAUSEDの間は点を追加しない(位置が信頼できないため)。マーカーが視界外(LOST)でもアンカーはVIOが維持しているので記録は継続する
- 保存先は `getExternalFilesDir()/route.json`。PCから `MSYS_NO_PATHCONV=1 adb pull /sdcard/Android/data/com.nacon01.kunekune/files/route.json` で取り出せる

## 実機検証(2026-07-21, Pixel 3a)

「マーカー正対→90度右を向く→立ち上がる→2m歩く」を記録した結果、9点・総距離2.46m。取り出したJSONの可視化で、(1)椅子を引く後退、(2)立ち上がりによる高さ+0.46m、(3)右方向への直線歩行1.2m、が順序・向き・距離すべて実際の動作と一致することを確認した。

可視化: https://claude.ai/code/artifact/dfa32076-eb76-4af3-999b-37e09955ab80

### 副産物として分かったこと

- マーカー座標系の軸の向き(壁掛けマーカーの場合): +x=マーカー正面から見て右、+y=マーカーから離れる方向、-z=上。立ち上がり動作が z の減少として現れたことから確認できた
- カメラを構える高さの変化(座り→立ち)は約0.5mで、これは軌跡の「高さ」成分にそのまま出る。経路の水平形状だけ使いたい場合は高さ成分を無視すればよい

## コード構成

- `RouteRecorder.kt` — サンプリング判定(0.3mルール)とポリライン構築。ARCore非依存でユニットテスト済み
- `RouteStore.kt` — route.json の保存・読込・削除(メタデータ: 記録日時・点数・総距離)
