# Phase 1d: 矢印誘導(pure pursuit)

## このフェーズで入った概念(1つだけ)

**「現在位置と保存済み経路の差分から進むべき方向を出す」**

保存済みポリラインに対して現在位置を射影し、少し先の経路上の点を目標として矢印を向ける。車のライントレースなどで使われる pure pursuit(純追跡)と同じ考え方。

## 誘導の計算(毎フレーム)

1. **射影**: 現在位置から経路の全区間への最短点を求め、経路上の弧長位置(スタートから何mの地点相当か)を得る
2. **lookahead**: 射影点から経路に沿って**1.0m先**の点を目標点とする。「今いる場所に一番近い点」ではなく「少し先」を狙うことで、経路の真上にいるときも進行方向を指し、カーブも滑らかに曲がれる
3. **角度差**: 目標点への水平ベクトルと、カメラの向き(-Z軸)の水平成分との符号付き角度。これが矢印の回転角になる
4. **残距離**: 射影点から終点までの弧長。**0.6m以内で「到着!」**
5. 矢印の表示角度は指数移動平均(EMA)で平滑化し、手ブレによる細かい揺れを吸収

## 座標系の設計判断

幾何計算はマーカー座標系ではなく**VIOワールド座標系(重力基準、+Yが上)**で行う。route.jsonのマーカー座標系の点は、毎フレーム現在のアンカーポーズでワールド座標に変換してから使う。

理由は2つ:
- 「水平面での方向」の計算が、マーカーが壁に貼られていても床に置かれていても正しくなる(マーカー座標系の軸は貼り方で意味が変わるため)
- アンカーはARCoreが継続補正しているので、毎フレームの変換が常に最新の位置合わせを反映する

## 実機検証(2026-07-21, Pixel 3a、2.5m経路)

- 矢印が記録済み経路の進行方向を正しく指す
- その場で体を回転させると矢印が追従する(角度差の計算が向きに正しく反応)
- わざと経路から外れると、経路へ戻る方向を指す(射影+lookaheadの効果)
- 終点0.6m以内で「到着!」表示

## 既知の制約

- **自己交差・近接往復する経路は不可**: 射影は「全区間から最近傍」を取るため、行きと帰りが近い経路では先の区間に飛びつく可能性がある。ソファ→風呂のような一方向経路では問題ない。必要になったら「前回の射影位置から前方に限定して探索する」改良で対処できる
- 誘導開始時に経路の始点から遠い場所にいると、最寄りの経路区間へ向かう方向を指す(始点へ誘導するわけではない)。運用上は「マーカーを映す=始点付近にいる」ので実害は小さい

## コード構成

- `GuidanceEngine.kt` — 射影・lookahead・角度差・到着判定の純粋ロジック。ARCore非依存、直線/L字経路でユニットテスト済み
- `GuidanceArrowView.kt` — Canvas描画の矢印View(EMA平滑化、トラッキング喪失時はグレー表示)
